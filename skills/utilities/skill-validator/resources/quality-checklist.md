# QUALITY CHECKLIST

USE THIS AS A FAST “DID I ACTUALLY DO THE WORK” PASS BEFORE YOU SHARE OUTPUTS OR MERGE CHANGES.

## CONTRACT AND SCOPE

- [ ] THE OUTPUT MATCHES THE SKILL’S DEFINED INPUT/OUTPUT CONTRACT
- [ ] ANY ASSUMPTIONS ARE EXPLICIT
- [ ] OUT-OF-SCOPE REQUESTS ARE HANDLED CLEANLY (REDIRECT, LIMITS, SAFETY)

## CLARITY

- [ ] INSTRUCTIONS ARE ACTIONABLE (VERBS + SPECIFIC NEXT STEPS)
- [ ] TERMS ARE DEFINED ON FIRST USE
- [ ] NO AMBIGUOUS PLACEHOLDERS (“DO THE THING”) WITHOUT EXAMPLES

## COMPLETENESS

- [ ] REQUIRED SECTIONS ARE PRESENT
- [ ] EDGE CASES ARE COVERED (AT LEAST 2–3)
- [ ] EXAMPLES INCLUDE BOTH INPUT AND EXPECTED OUTPUT

## CONSISTENCY

- [ ] HEADERS ARE ALL CAPS (`#`, `##`, `###`)
- [ ] NAMING AND TERMINOLOGY ARE CONSISTENT ACROSS DOCS
- [ ] LINKS AND FILE PATHS ARE CORRECT

## USABILITY

- [ ] A NEW PERSON COULD FOLLOW THIS WITHOUT INSIDER CONTEXT
- [ ] COPY/PASTE WORKS (NO HIDDEN DEPENDENCIES)
- [ ] STEPS ARE ORDERED (NO “DO STEP 7 BEFORE STEP 2”)

## CHANGE HYGIENE

- [ ] `CHANGELOG.md` UPDATED UNDER “UNRELEASED”
- [ ] `TESTING-NOTES.md` UPDATED WITH WHAT YOU VERIFIED
